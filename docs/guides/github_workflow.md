# Документация по работе с GitHub

Этот документ описывает базовый рабочий процесс в репозитории Gifty: получение доступа, настройка локальной среды, правила ветвления и pull‑request.

## Получение доступа

Доступ к репозиторию можно получить:

- через запрос в телеграм‑боте,
- либо через добавление вашим ментором.

После получения доступа проверьте, что у вас есть права на чтение и запись в репозиторий.

## Настройка локальной среды

1. Установите Git.
2. Настройте авторство:

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "you@example.com"
```

3. Клонируйте репозиторий:

```bash
git clone git@github.com:IceDarold/gifty-backend.git
```

Если SSH недоступен, используйте HTTPS‑URL.

## Рабочий процесс

### 1) Создание ветки

Всегда работайте в отдельной ветке:

```bash
git checkout -b feature/<краткое-описание>
```

### 2) Коммиты

Делайте небольшие, логически цельные коммиты. В описании коммита указывайте суть изменения.

### 3) Pull Request

1. Запушьте ветку в GitHub:

```bash
git push origin feature/<краткое-описание>
```

2. Откройте Pull Request в GitHub.
3. В описании PR укажите:

- контекст задачи;
- что было сделано;
- как проверялись изменения.

### 4) Ревью

- Ответьте на комментарии ревьюеров.
- Не сливайте PR без одобрения.

## Синхронизация с основной веткой

Регулярно подтягивайте изменения из основной ветки:

```bash
git fetch origin
git merge origin/main
```

Если в проекте используется другая основная ветка (например, `master`), используйте ее.

## Безопасность и доступы

- Не коммитьте секреты и ключи.
- Не публикуйте данные окружений в открытых файлах.
