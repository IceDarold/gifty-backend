# Gift Query & Feedback Strategy

> **"Мы не спрашиваем абстрактно. Мы калибруем 5 Dimensions."**

Этот документ описывает логику сбора информации о пользователе (Quiz) и динамического уточнения предпочтений (Feedback Loop). Цель — построить профиль на основе [Grand Unified Theory of Gifting](grand_unified_theory.md).

---

## 0. Модель профиля

Система должна определить:
1.  **Веса измерений (0–100):** Mirror, Optimizer, Catalyst, Anchor, Permission.
2.  **Доминирующие подтипы:** (например, *Mirror: Fanatic* или *Optimizer: Upgrade*).
3.  **Жесткие ограничения:** Бюджет, анти-категории, аллергии, уровень минимализма.

---

## 1. Onboarding Quiz (Static Profiling)

**Формат:** 12–16 вопросов. Всегда есть вариант "Не знаю".

### Module 1: Context (Калибровка)
*Цель: определить границы допустимого (Anchor/Catalyst).*

1.  **Кто он вам?**
    *   *Варианты:* Коллега / Знакомый / Друг / Близкий друг / Партнёр / Родственник / Родитель.
    *   *Влияние:* Отсекает слишком личные (Anchor) или требующие усилий (Catalyst) подарки для дальних кругов.
2.  **Повод и тон:**
    *   *Варианты:* Формально / Тепло и заботливо / Романтично / С юмором / "Вау-эффект".
    *   *Влияние:* "Формально" → Permission/Optimizer. "Тепло" → Anchor/Mirror.
3.  **Бюджет и сроки:**
    *   *Варианты:* Диапазон + "Нужно сегодня / На неделе / Не спешу".

### Module 2: Mirror (Identity)
*Цель: Найти зацепки за личность (Fanatic, Aesthetic, Tribe).*

4.  **Стабильные черты личности** (выбери 2–3 тега):
    *   *Варианты:* Эстет, Практик, Интроверт, Технарь, Творческий, Спортсмен, Домосед, Путешественник, Гик, Душа компании.
    *   *Влияние:* Базовый вектор Personal Myth.
5.  **Текущие увлечения / Фандомы:**
    *   *Ввод:* (Поле для ввода 1–3 увлечений, например: "Гарри Поттер, Кофе, Теннис").
    *   *Влияние:* Прямой триггер для Mirror: Fanatic.
6.  **Стиль и Эстетика ("Вайб"):**
    *   *Варианты:* Минимализм / Уют (Cozy) / Классика / Ярко и смело / Технологично / Винтаж / Не знаю.
    *   *Влияние:* Критично для Mirror: Aesthetic и Home Decor.
7.  **Придирчивость к деталям (Micro-preferences):**
    *   *Вопрос:* "Он очень избирателен в..."
    *   *Варианты:* Еда/Кофе / Музыка / Канцелярия / Гаджеты / Одежда / Дом / Ничего такого.
    *   *Влияние:* Если выбрано → ищем Mirror: Taste Token. Если "Ничего" → избегаем специфичных вещей.

### Module 3: Optimizer (Friction)
*Цель: Найти бытовую боль (Upgrade, Fix, Automation).*

8.  **Точки боли** (выбери до 2):
    *   *Варианты:* Время / Порядок / Усталость / Забывчивость / Стресс / Поездки / Быт / Сон.
    *   *Влияние:* Определяет домен (System, Comfort, Buffer).
9.  **Стиль решения проблем:**
    *   "Лучше качество того, что уже есть" (→ *Upgrade*)
    *   "Убрать конкретную раздражалку" (→ *Fix*)
    *   "Навести систему/порядок" (→ *System*)
    *   "Пусть само делается" (→ *Automation*)
10. **Готовность к настройке (Tech Tolerance):**
    *   "Поставил и работает" (Plug & Play).
    *   "Немного настроить ок".
    *   "Любит возиться и копаться".
    *   *Влияние:* Фильтр для гаджетов и умного дома.

### Module 4: Catalyst (Potential)
*Цель: Поддержать стремления (Starter, Horizon).*

11. **Амбиции ("Хочу, но не начал"):**
    *   *Варианты:* Язык / Спорт / Творчество / Готовка / Карьера / Публичность / "Что-то новое" / Нет.
    *   *Влияние:* Вектор для Catalyst.
12. **Стадия:**
    *   "Только думает/мечтает" (→ *Starter*)
    *   "Уже пробует иногда" (→ *Commitment*)
    *   "Занимается регулярно/профи" (→ *Accelerator*)
13. **Отношение к усилиям:**
    *   Любит вызов (Challenge) / Нейтрально / Не любит "подарки-домашку".

### Module 5: Anchor (Connection)
*Цель: Оценить глубину связи.*

14. **Уровень личного:**
    *   *Шкала 0–10:* От "Только нейтрально" до "Очень личное/интимное ок".
15. **Общий контекст:**
    *   *Вопрос:* "Есть ли у вас локальные шутки, общая история?" (Да/Нет + поле для контекста).
    *   *Влияние:* Inside Language / Time Capsule.

### Module 6: Permission (Guilt)
*Цель: Разрешить удовольствие.*

16. **Самостоятельные покупки:**
    *   "Покупает себе всё сам" (→ *Permission* нужен сильный).
    *   "Иногда балует себя".
    *   "Почти никогда не тратит на себя" (→ *Permission* критичен).
17. **Тип удовольствия (Ранжирование топ-2):**
    *   Уют/Комфорт / Вкусное / Красивое / Игра/Фан / Тихий премиум / Статус.

## 2. Dynamic Feedback Loop (Smart Scroll)

Принцип: **"1 Клик → 1 Микро-вопрос"**. Мы спрашиваем *почему* рекомендация зашла или нет.

### Event A: Like (👍) — "Что именно зацепило?"
*   "Стиль / Эстетика" → `Mirror: Aesthetic` boost.
*   "Прямо про него" → `Mirror: Personal Myth` boost.
*   "Полезно / Удобно" → `Optimizer` boost (далее уточнить: Fix или Automation?).
*   "Милое / Про нас" → `Anchor` boost.
*   "Он бы сам не купил, но кайф" → `Permission` boost.
*   "Это для роста / хобби" → `Catalyst` boost.

### Event B: Dislike (👎) — "Почему нет?"
*   "Не его стиль" → Сменить вектор `Aesthetic`.
*   "Бесполезно" → Снизить `Permission`, повысить `Optimizer`.
*   "Слишком личное" → Ограничить `Anchor`.
*   "Слишком сложно" → Ограничить `Catalyst`.
*   "Уже есть" → Предложить `Optimizer: Upgrade`.
*   "Выглядит как хлам" → Включить фильтр минимализма.
*   "Сомневаюсь в качестве/бренде" → Фильтр `Trusted Brands` для Optimizer.

### Event C: Doubt (🤔) — "Что мешает?"
*   Цена / Доставка / Слишком банально / Слишком рискованно.

## 3. Smart Questioning Logic

Система задает вопросы не случайно, а "бьет" в зоны максимальной неопределенности.

### Шаг 1. Определение доминанты
Если пользователь 3 раза лайкнул "Полезные" товары, система понимает: *Доминирует Optimizer*.

### Шаг 2. Развилка (Fork)
Система задает уточняющий вопрос внутри доминанты:
*   **Если Optimizer:** "Ему важнее: Качество (Upgrade), Порядок (System) или Автоматизация?"
*   **Если Mirror:** "Это про: Увлечение (Fanatic), Стиль (Aesthetic) или Ценности (Values)?"
*   **Если Permission:** "Ему приятнее: Уют, Вкус, Красота или Игра?"

---

## 4. Safety Filters (Быстрые шкалы)

Эти параметры можно выяснить в Quiz или через первые 5 карточек, чтобы избежать провалов:

1.  **Минимализм vs Вещизм**: Фильтр "Хлама" (Clutter).
2.  **Сентиментальность (0–10)**: Фильтр "Пылесборников" (Anchor).
3.  **Risk Tolerance**: Любит сюрпризы vs Только по вишлисту.
4.  **Практичность**: Насколько подарок должен быть применим (Permission vs Optimizer).
