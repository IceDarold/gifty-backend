    Gift Query & Feedback Strategy

    "100% точность недостижима. Наша цель — максимизировать вероятность и явно показать уверенность."

    Этот документ описывает полную архитектуру сбора данных для построения профиля пользователя на основе Grand Unified Theory of Gifting.

    1. Архитектура сбора данных (3 слоя)

    Слой 1 — Интерактивный Опрос (Core Quiz)

    • Ветвящаяся структура: 12–18 обязательных вопросов + 10–25 ситуативных.
    • Каждый ключевой ответ имеет параметр Confidence (Знаю точно / Кажется / Не знаю).
    • Цель: Построить базовый вектор весов измерений.

    Слой 2 — Active Learning (Scroll Loop)

    • 1 Micro-Question после каждого действия (Like/Dislike).
    • Pairwise Choice каждые 6–8 карточек (A или B?).
    • Система выбирает вопрос, который максимально снижает энтропию профиля.

    Слой 3 — External Verification (Optional)

    • Анализ соцсетей/вишлистов/скриншотов (с явного согласия).
    • Используется для верификации Mirror (фандомы, эстетика) и Catalyst (мечты).

    2. Модель профиля (System Entities)

    Результатом работы является структурированный отчет:

    1. 5 Dimensions (Weights 0-100 + Confidence 0-1):
      • Mirror, Optimizer, Catalyst, Anchor, Permission.
    2. Subtypes (Weights 0-100):
      • Например: Mirror: Aesthetic (80%), Optimizer: Automation (65%).
    3. Constraints & Risks:
      • Hard Limits (аллергии, категории "нет").
      • Soft Limits (минимализм, "без хлама").
      • Context (совместимость iOS/Android, размеры).
    4. Signal Evidence List:
      • Конкретные бренды, цитаты, боли, мечты.

    3. Детальная структура Опроса (Quiz Logic)

    Блок 0. Контекст (Обязательный)

    1. Кто вы ему? (Коллега / Приятель / Друг / Близкий друг / Партнёр / Родственник / Родитель).
      • Logic: Если близость ≤ 2 → Anchor limit, Catalyst только "Starter".
    2. Повод: (ДР / Праздник / Спасибо / Извинение / "Просто так" / Событие).
    3. Близость (1–5): Оценка уровня интимности.
    4. Бюджет: Диапазон + чекбокс "Только exact price" или "Можно +30% за мэтч".
    5. Срок: Дата + Формат (Лично / Доставка / Цифровой).
    6. Возрастной диапазон: (15–18 / 18–25 / 25–35 / ...).
    7. Пол / Гендер: (Опционально, для фильтрации одежды/парфюма).

    Блок 1. Жесткие ограничения (Safety First)

    1. Точно НЕ дарить (Мультивыбор):
      • Одежда/Обувь, Парфюм/Косметика, Еда/Алкоголь, Сертификаты, Декор/Сувениры, Шутки, Личное/Интимное, Гаджеты.
    2. Здоровье/Этика: Аллергии, Диеты, Веганство, Эко-принципы.
    3. Отношение к вещам:
      • Любит вещи / Нейтрально / Минимализм / Ненавидит хлам.
    4. Пространство: Мало места / Норм / Много места (для габаритных подарков).
    5. Техника: Экосистема (iOS/Android) + "Есть ли AirTag-аналог?".
    6. Этические фильтры: Без кожи, Локальные бренды, Zero Waste.

    Блок 2. Mirror (Identity Map)

    Мы собираем наблюдаемые сигналы, а не абстракции.

    1. Топ-3 Роли/Тега (Выбор + Свой вариант):
      • Эстет, Технарь, Гурман, Спортсмен, Геймер, Меломан, Читатель, Путешественник, Домосед, Тусовщик, Человек-система, Творец, Коллекционер, Карьерист.
      • Confidence: Знаю точно / Кажется / Не уверен.
    2. Топ-3 Темы разговоров: О чем может говорить часами?
      • Уточнение: Это давно или временное увлечение?

    Блок 3. Fanatic (Obsessions)

    1. Есть ли 1–2 обсессии? (Да/Нет).
      • Если да: Что именно? (Фандом / Игра / Артист / Бренд).
      • Level: Casual / Фанат / Коллекционер.
      • Negative Filter: Что точно НЕ любит внутри темы? (Не этот сезон / Не этот персонаж).
      • Check: "У него уже есть мерч? Какой?" (Редкое vs Массовое).

    Блок 4. Aesthetic (Visual Language)

    1. Выберите 2 эстетики:
      • Минимализм, Уют (Cozy), Винтаж, Techwear, Арт, Спорт, "Дорого-тихо", Ярко-максимализм, Кавай/Cute, Индастриал, Натуральность.
    2. Палитра: Светлое / Тёмное / Нейтральное / Яркое.
    3. Материалы: Металл / Дерево / Керамика / Ткань / Кожа / Пластик.
    4. Логотипы: Любит брендинг / Только no-name.
    5. Проверка: Ему важнее как выглядит или как работает? (Slider 0-10).

    Блок 5. Taste Token (Micro-Preferences)

    1. Daily Drivers (Что потребляет ежедневно?):
      • Кофе, Чай, Вода, Снеки, Уход, Музыка, Канцелярия.
    2. Точные "Люблю/Не люблю":
      • Вкусы (Сладкое/Горькое), Запахи (Цитрус/Ваниль), Текстуры.
    3. Спецификация: Подарок должен быть "точным по модели" или "сюрприз ок"?

    Блок 6. Values Signal (Principles)

    1. Проявленные ценности (до 3):
      • Качество/Долговечность, Практичность, Статус, Эмоции, Здоровье, Творчество, Технологии, Экология.
    2. Принципиальные "НЕТ": Из-за чего он не купит вещь?

    Блок 7. Tribe (Badge)

    1. Где он "свой"?
      • Бег/Зал, Настолки, Киберспорт, Стартап, Кино/Арт, Фото, Музыка, Авто.
    2. Level: Гик по деталям vs Любитель.

    Блок 8. Personal Myth (Archetype)

    1. Архетип: Исследователь, Создатель, Куратор, Перфекционист, Защитник, Комик.
    2. Self-Worth: Что он в себе ценит? (Ввод 1-2 предложения).

    Блок 9. Optimizer (Friction)

    1. Топ-3 раздражителя в быту: (Ввод).
    2. Стиль: Терпит неудобства vs Оптимизирует всё.
    3. Зоны боли:
      • Зарядка/Провода, Ключи, Сон/Стресс, Рабочее место, Уборка, Здоровье/Спина.
    4. Risk Check: Есть ли темы, где фикс выглядит как упрёк? (Вес, Гигиена, Порядок).

    Блок 10. Catalyst (Potential)

    1. Мечты ("Хочу начать"): 1–3 пункта (Язык, Спорт, Хобби).
    2. Reality Check: "Говорит и делает" vs "Говорит и откладывает".
    3. Learning Style: Курсы / Практика / Комьюнити / Самоучка.
    4. Время на хобби: Мало / Средне / Много.

    Блок 11. Anchor (Bond)

    (Только если Близость ≥ 3)

    1. Shared Context: Есть ли общий символ/история?
    2. Sentimentality: Любит сентиментальность vs "Не надо соплей".
    3. Intimacy: Принимает ли очень личные подарки?

    Блок 12. Permission (Guilt-Free)

    1. Self-Spending: Отказывает себе? (Часто / Иногда / Никогда).
    2. Guilty Pleasures:
      • Вкусное, Запахи, Ванна/Уход, Ткани/Пижама, Декор, Премиум-аксессуары.
    3. Preference: "Тихо-качественно" vs "Вау-эффект".

    4. Динамика во время скролла (Active Learning)

    Event A: Like (👍) — "Что зацепило?" (1 tap)

    • Стиль/Красиво → Mirror: Aesthetic boost.
    • Прямо "про него" → Mirror: Identity boost.
    • Полезность → Optimizer boost.
    • Поддержка мечты → Catalyst boost.
    • Тепло/Лично → Anchor boost.
    • Сам бы не купил → Permission boost.
    • Smart Fork: Если выбран Optimizer → "Ему важнее качество (Upgrade) или чтобы само работало (Automation)?"

    Event B: Dislike (👎) — "Почему нет?"

    • Не его стиль → Сменить Aesthetic.
    • Бесполезно → Permission down / Optimizer up.
    • Слишком сложно → Catalyst risk.
    • Уже есть → Предложить Upgrade.
    • Хлам → Minimalism filter.
    • Слишком лично → Anchor limit.

    Event C: Pairwise (A vs B)

    Каждые 8 карточек: * Практично vs Эстетично? * Вещь vs Опыт? * Личное vs Нейтральное?

    Stop Rule: Остановка вопросов, когда веса измерений стабилизировались (Delta < X).

    5. Механика "Точности"

    1. Confidence Weights:
      • Сильные сигналы (Explicit "Знаю точно", Конкретные бренды) = x2..x4 вес.
      • Слабые сигналы (Implicit, "Кажется") = x1 вес.
    2. Consistency Check:
      • Конфликт "Минимализм" + "Любит сувениры" → Задать уточняющий вопрос.
      • Конфликт "Не любит сертификаты" + "Хочет опыт" → Уточнить формат ("Билет > Ваучер").
    3. Negative Filtering:
      • Триггеры отторжения имеют наивысший приоритет (Vetoe Power).

    6. Финальный Отчёт (Output Concept)

    User Portrait: * Dominant: Mirror (78%), Optimizer (64%). * Subtypes: Aesthetic (Minimalism), TasteToken (Coffee), Optimizer (System). * Red Flags: NO clutter, NO logos, iOS only. * Evidence: [Любит кофе спешиалти, работает из дома, жалуется на бардак на столе].

    Strategy Recommendations: 1. Strategy 1 (The Upgrade): Премиум-аксессуар для рабочего стола (Mirror + Optimizer). 2. Strategy 2 (The Ritual): Набор для альтернативного заваривания (Mirror + Permission). 3. Strategy 3 (The Fix): Органайзер кабелей (Optimizer + Aesthetic).

    Framing: "Я заметил, как ты ценишь порядок на столе, поэтому..."
